# MenuCaptain — change recap (2026-06-28)

Four changes made in a parallel Claude Code session. **All are already written into the
working files (uncommitted). None are deployed.** Verified to compile/mount; the signed-in
screens are behind auth so they need on-device testing after deploy.

## Front-end — `C:\Users\cjgra\dining-log-app\index.html`

1. **Local-date fix.** `todayISO()` was `new Date().toISOString().slice(0,10)` (UTC), which
   made visit/receipt dates roll to *tomorrow* in the evening. Now builds from local
   `getFullYear/getMonth/getDate`. `created_at`/`updated_at` timestamps left as UTC on purpose.

2. **Nutrition block moved.** The meal-level "Nutrition (estimate from a photo)" `<div className="fld">`
   in `VisitForm` moved from up by Photos to just **below the dishes / "Tap from menu"** section
   (right before Notes), so the whole-plate estimate sits next to the food.

3. **Inline Google finder in the visit restaurant picker.** When saved places don't match,
   the picker now shows **Search Google** (`placesSearchText`) + **Near me** (geolocation →
   `placesNearby`). Tapping a result creates + selects the place inline via a new
   `onCreateRestaurant(rec)` prop (app shell prepends + commits; `uid("rest")`, name,
   `google_place_id`, address, lat/lng). Self-contained in `VisitForm` so the visit draft is
   preserved. Pro-gated server-side (402 → friendly copy).

4. **Restaurant Google photo.** New `placesPhoto(cfg, name, maxwidth)` helper; `updateFromGoogle`
   now also stores `google_photo_name` + `google_photo_attr` on the restaurant; new `GooglePhoto`
   component renders the place photo as the `.rest-hero` **with attribution** when there's no
   uploaded `r.photo`. Photo is fetched on demand, never stored (Google terms).

## Backend — `C:\Users\cjgra\dining-captain-backend\main.py`

5. **Places photo support.**
   - Added `"photos"` to `PLACES_DETAILS_FIELDS`.
   - `places_details` now returns `photo_name` (Google resource name `places/{id}/photos/{ref}`)
     + `photo_attribution` (first authorAttribution displayName).
   - Added `PLACES_COST["photo"] = 0.007`.
   - New `GET /api/places/photo?name=&maxwidth=` — Pro-gate + meter + circuit-breaker (mirrors
     the other Places endpoints), calls Google `{name}/media?...&skipHttpRedirect=true`, returns
     `{url: photoUri}` (short-lived, cached 30 min, never stored; validates `name` starts with
     `places/` and contains `/photos/`).
